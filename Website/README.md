# LibreLoom's Logo and Other Related Things

**Refer to this page for our style and branding guidelines.**

### We use the Simplex Mono design language.

Simplex Mono design guidelines:

- Color Palette:
  - White (Default Primary): #FFFFFF
  - Black (Default Secondary): #000000
  - Grey (Default UI accent): #767676
  - There must be a light/dark mode toggle which just swaps primary and secondary colors by default.
- Typography:
  - Headings: FreeMono Regular-Weight (or similar monospace font like Courier New)
  - Main Body Text: Noto Sans Regular-Weight (or similar sans-serif font like Helvetica Neue)
  - Accents: Use italics for single or two word emphasis. Use bold for emphasis on entire lines.
- Overall UI design:
  - Flat design with lots of padding (enough to look not compressed but also doesn't feel big.)
  - Minimal visual effects & shadows (like, really, really minimal, preferably zero).
  - Circle/pill/fully-rounded-rectangle UI elements. Larger elements may be rounded rectangles.
  - Navigation/toolbars float, and go on the *bottom*. They collapse to circular hamburger menu buttons on small screens, and can then be dragged/flinged (like the selfie view in Apple's dual camera recording) to any corner of the screen.
  - Sections in nav/toolbars are separated with "|"-like shapes. The divider uses the accent color and almost spans the navbar's full height.
  - Nice, fluid animations. Take inspiration from Google's Material Expressive V3 for this.
  - Well-tuned, noticeable haptics on supported platforms (e.g., Android, iOS, Linux Mobile, web if you can make it work *well*). A toggle must be available to enable or disable haptics. Offering intensity control is optional but recommended.
- Customization:
  - The primary, secondary, and accent must be able to be set by the user.
  - There must be a "use different color palette for dark mode" option.
